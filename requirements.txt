# What JARVIS actually needs at runtime on this PC:
claude-agent-sdk        # the Claude brain (uses your Claude subscription)
pyttsx3                 # offline text-to-speech (falls back to Windows System.Speech)

# Everything else (UI, PC control, voice input) uses only the Python standard
# library + built-in Windows components, so there are no other dependencies.
#
# NOTE: faster-whisper / sounddevice / openwakeword were evaluated for
# microphone speech-to-text but are NOT used here — this PC's Smart App Control
# blocks their native libraries, and its audio stack exposes no working capture
# device. Voice input instead uses the OS recognizer (see voice_input.py).
